記事一覧・詳細 — 関連記事クラスタ UI モック

画面上は階層とインデントだけで塊を表し、一覧ページと記事詳細で同じマークアップ・同じ見た目のブロックを使います。データ上の親子や parent_slug などの話は、下の設計メモに閉じます。

設計メモ（画面には出さない）

先頭行＝塊の入り口となる記事、インデントした行＝その記事にぶら下がる関連稿、というモデルで実装できます。詳細ページでは「この記事と同じ塊の一覧」をそのまま出し、 aria-current="page" 相当の行に .is-current を付けて背景と左線を強めます。

配置の目安：文頭にこのブロックがあると、流入直後に塊の文脈が掴めます。文末にも同じブロックを置くと読了後の移動に便利です。両方とも同一コンポーネントでよく、文言や別パターンのリボンは不要です。
